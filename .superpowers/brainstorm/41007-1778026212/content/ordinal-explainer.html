Ordinal Regression for Stabilizing Steps

Why not plain regression or plain classification?

The Problem with Each Approach

MSE Regression (current)

Assumes equal spacing: predicts 2.1 when the answer is either 0 or 5. Optimizes for "close on average" which means almost never correct for a long-tailed distribution.

Classification (K-way softmax)

Treats 0, 1, 2, ..., 43 as unrelated categories. Ignores that 4 is closer to 5 than to 40. Data-hungry: needs examples of every class.

Ordinal Regression ★

Hits the sweet spot. Knows that 2 > 1 but doesn't assume the gap is the same as 10 → 11.

How Ordinal Regression Works

Instead of predicting a value directly, predict K-1 binary decisions:

P(stab_steps > 0) — "Is it non-zero?" (38% are 0)

P(stab_steps > 1) — "Is it more than 1?"

P(stab_steps > 2) — "More than 2?"

...

P(stab_steps > 42) — "More than 42?"

One forward pass gives all K-1 probabilities simultaneously (one Linear head with K-1 outputs). By construction: P(>0) ≥ P(>1) ≥ P(>2) ≥ ... because they share weights with learned biases.

Inference: Simple

Output: [0.92, 0.85, 0.60, 0.31, 0.05, 0.00, ...]

These mean: P(>0)=0.92, P(>1)=0.85, P(>2)=0.60, P(>3)=0.31, ...

Prediction: count thresholds where P > 0.5 → 3

 

The first threshold P(>0) acts as an automatic zero-gate: if the block is trivial, P(>0) will be low and output is 0.

Why It Fits This Data

38.2% of samples are 0 — the ordinal gate naturally learns "if block_size=1, always predict 0"

Long tail up to 43 — rare high values don't need their own training examples; they're just progressively smaller probabilities

Conditional on block_size — larger block → each threshold shifts higher. The model learns this as a simple shift in the latent representation feeding the ordinal head.

Loss: Binary cross-entropy summed over all K-1 thresholds. Each training example contributes to every threshold — so every example teaches the model about multiple decision boundaries.
