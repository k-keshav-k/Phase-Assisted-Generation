Data Patterns Found in GSM8K Trace Tuples

94,911 tuples · 5,000 training sequences · Key patterns your current model doesn't capture

1. Block Size is Trimodal, Not Normally Distributed

MSE regression predicts the conditional mean. But the true distribution has three distinct regimes:

block_size=1 ████████████████████████████████████████ 19.4%

(phase boundary marker — always max_stab_step=0)

block_size=16 ████████████████████████████████████████████████████████████████ 28.2%

(AdaBlock default — most common content block)

block_size=2-15, 17-64 ≈ remaining 52%

(other content blocks, roughly geometric tail)

Consequence: MSE regression predicts ~8-9, which is never correct for either the boundary (1) or default (16) regime.

2. Stabilizing Steps are Nearly Determined by Block Size

block_size=1 → max_stab_step=0 (100% of cases)

block_size=2 → max_stab_step=0 (74.7%), mean=0.25

block_size=8 → max_stab_step=0 (41.7%), mean=1.56

block_size=16 → max_stab_step=0 (6.1%), mean=5.72, median=6

block_size=27 → max_stab_step=0 (12.8%), mean=3.81

Monotonic relationship: larger blocks → more stabilisation steps. Predicting stab_steps without knowing block_size is pointless — the conditional structure is strong.

3. Sequential Structure: Alternating Boundary-Content

tokens: [content block (size 23)] → [BOUNDARY (size 1)] → [content block (size 16)] → ...

 

Top-3 transitions:

1 → 16: 8,114 occurrences (boundary → default block)

16 → 16: 6,645 (consecutive default blocks)

16 → 1: 1,860 (content → boundary)

93.9% of block_size=1 occurrences are preceded by a block > 1. The model should learn this structural pattern.

Core Question

Given these patterns, the current regression approach is fundamentally mismatched. The likely solution combines:

  • Classification for block_size (discrete choice among 64 values)
  • Conditional prediction of stab_steps given block_size
  • Leveraging the alternating sequence structure (boundary ↔ content)
